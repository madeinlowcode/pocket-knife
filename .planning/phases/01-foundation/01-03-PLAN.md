# Pocket-Knife Setup

Guide the user through creating or updating `~/.claude/.env` with their API keys.

## Instructions

When this skill is invoked, follow these steps:

### Step 1: Check existing configuration

Read `~/.claude/.env` if it exists:
- If it exists, show which providers are already configured (show key names only, NEVER show key values)
- If it does not exist, explain that you will create it

### Step 2: Ask what the user wants to configure

If no argument was provided, ask the user which provider categories they want to set up:

```
Which categories would you like to configure?
1. Image (Google Gemini, fal.ai FLUX) — needs: GOOGLE_API_KEY, FAL_KEY
2. Audio (ElevenLabs TTS/STT) — needs: ELEVENLABS_API_KEY
3. Video (Google Veo, fal.ai) — needs: GOOGLE_API_KEY, FAL_KEY
4. LLM (Grok, Kimi, GLM) — needs: XAI_API_KEY, KIMI_API_KEY, GLM_API_KEY
5. Search (Tavily, Exa) — needs: TAVILY_API_KEY, EXA_API_KEY
6. All of the above
```

If an argument was provided (e.g., `/pocket-knife:setup image`), configure only that category.

### Step 3: Collect keys for the selected categories

For each required key, ask the user to provide it. Check first:
1. `[ -n "$KEY_NAME" ]` — key already in environment (loaded from previous .env)
2. `grep -q "KEY_NAME" ~/.claude/.env 2>/dev/null` — key already in file

If the key already exists, ask: "PROVIDER_KEY is already configured. Update it? (y/N)"

When asking for a key:
- Tell the user WHERE to get it (URL to provider dashboard)
- Confirm you will write it to ~/.claude/.env
- NEVER print or echo the key value back to the user

### Step 4: Write to ~/.claude/.env

After collecting keys, write them to `~/.claude/.env`.

For a new file, use Write tool to create:
```
# Pocket-Knife API Keys
# Generated by /pocket-knife:setup — DO NOT COMMIT THIS FILE
# Provider keys — get yours at the provider's dashboard

GOOGLE_API_KEY=<value>
ELEVENLABS_API_KEY=<value>
# ... (only keys user provided)
```

For an existing file:
- Read the current content first
- Append only new keys (do not overwrite existing ones unless user confirmed)
- Preserve comments and formatting

### Step 5: Confirm and suggest reload

After writing:
1. Show key names that were saved (not values)
2. Say: "Run `/reload-plugins` to make these keys available in the current session."
3. Suggest testing with the relevant skill

- Never run `cat ~/.claude/.env` or print key values
- If you suspect a key was leaked, revoke it on the provider dashboard and set a new one here
- This file is read once per session at startup via the SessionStart hook
