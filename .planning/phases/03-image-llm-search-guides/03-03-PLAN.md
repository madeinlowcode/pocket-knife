# Prompt Engineering Guide

## LLM Prompts

**Formula:** Role + Task + Constraints + Output Format

| Element | Example |
|---------|---------|
| Role | "You are a senior software engineer specializing in security" |
| Task | "Review this code for vulnerabilities" |
| Constraints | "Focus on SQL injection and XSS only" |
| Output | "Return a JSON list of issues with severity: critical/high/medium/low" |

**Techniques:**

| Technique | When to use | Example |
|-----------|-------------|---------|
| Chain-of-thought | Complex reasoning | "Think step by step before answering" |
| Few-shot | Pattern matching | Provide 2-3 examples before the actual request |
| System prompt | Persistent behavior | Set tone, persona, format at session start |
| Temperature 0 | Deterministic output | Code generation, data extraction |
| Temperature 0.7+ | Creative output | Brainstorming, marketing copy |

**Chain-of-thought template:**
```
Let's solve this step by step:
1. First, understand the problem: [restate in your own words]
2. Identify constraints: [list limitations]
3. Consider approaches: [list 2-3 options]
4. Choose the best approach: [explain why]
5. Execute: [the actual answer]
```

## Image Generation Prompts

**Formula:** Subject + Style + Composition + Lighting + Technical

| Element | Examples |
|---------|----------|
| Subject | "portrait of a scientist", "mountain landscape", "product shot of a coffee cup" |
| Style | "photorealistic", "watercolor", "cinematic", "oil painting", "digital art" |
| Composition | "centered", "rule of thirds", "close-up", "wide shot", "bird's eye view" |
| Lighting | "golden hour", "studio lighting", "dramatic shadows", "soft diffused light" |
| Technical | "4K", "high resolution", "sharp focus", "bokeh background", "HDR" |

**Complete example:**
`"Portrait of a scientist in a lab, oil painting style, centered composition, warm laboratory lighting, detailed, high resolution"`

**Negative prompts** (what to exclude):
`"blurry, low quality, distorted, extra limbs, watermark, text"`

## Video Prompts

**Formula:** Shot Type + Subject + Action + Setting + Lighting + Style + Technical

| Shot Type | Use for |
|-----------|---------|
| Tracking shot | Following moving subjects |
| Static/locked | Dialogue, product reveals |
| Aerial/drone | Establishing shots, scale |
| Close-up | Emotion, product details |
| POV | Immersive first-person |

**Complete example:**
`"Tracking shot of a red sports car, driving through Tokyo at night, neon reflections, cinematic color grade, 4K 24fps"`

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Too vague: "write code" | Specify language, purpose, constraints: "Write Python function to validate email format, return bool" |
| Too long: 500-word prompt | Break into steps; use few-shot for structure |
| No output format | Always specify: "Return as JSON", "Answer in bullet points", "Max 3 sentences" |
| Missing context | Include relevant background: tech stack, audience, goal |
| Image: just the subject | Add style + lighting + technical for quality |

## Prompt Templates by Use Case

**Code review:** "Review [LANGUAGE] code for [CONCERN]. Focus on [SPECIFIC_ASPECT]. Return issues as: [ISSUE]: [EXPLANATION] ([SEVERITY])"

**Content creation:** "Write a [FORMAT] about [TOPIC] for [AUDIENCE]. Tone: [TONE]. Length: [LENGTH]. Include: [KEY_POINTS]"

**Data analysis:** "Analyze this [DATA_TYPE] and identify [GOAL]. Format: [FORMAT]. Highlight anomalies above [THRESHOLD]"

# Design Guide

## Logo Design

**Process:**
1. Define brand personality (3 adjectives: e.g., "bold, modern, trustworthy")
2. Choose style: wordmark, lettermark, icon+text, abstract icon
3. Select color palette (2-3 colors max for logos)
4. Test at small sizes (16px favicon) and large sizes

**Color psychology:**
| Color | Associations | Industries |
|-------|-------------|------------|
| Blue | Trust, stability | Finance, tech, healthcare |
| Green | Growth, nature | Sustainability, health, finance |
| Red | Energy, urgency | Food, retail, entertainment |
| Black | Luxury, sophistication | Fashion, premium products |
| Orange | Creativity, warmth | Food, startups, media |

**Logo prompt template for AI:**
`"Minimalist [STYLE] logo for [BRAND], [COLOR1] and [COLOR2] color palette, clean lines, vector style, white background, professional"`

## OG Image Design (Social Share)

**Specs:** 1200x630px, under 8MB, JPG or PNG

**Layout formula:**
```
[BACKGROUND: brand color or image]
[LOGO: top-left, max 20% width]
[HEADLINE: center, 60-80px, high contrast]
[SUBHEADLINE: below headline, 30-40px]
[CTA or URL: bottom, smaller]
```

**Common mistakes:**
- Text too close to edges (safe zone: 60px from each edge)
- Low contrast (test with grayscale)
- Too much text (max 6-8 words for headline)
- Wrong aspect ratio (platforms crop differently)

## Landing Page Design

**Above-the-fold checklist:**
- [ ] Clear headline stating the value proposition (what, for whom)
- [ ] Subheadline with supporting detail (how, proof point)
- [ ] Primary CTA button (high contrast, action verb: "Start Free", "Get Demo")
- [ ] Hero image or product screenshot
- [ ] Social proof (logos, numbers, testimonial snippet)

**Section order:**
1. Hero (value prop + CTA)
2. Problem (pain point)
3. Solution (product/service)
4. Social proof (testimonials, logos, metrics)
5. Features (how it works)
6. Pricing (if applicable)
7. FAQ
8. Final CTA

## YouTube Thumbnail Design

**Formula:** Emotion face + text overlay + bold colors

| Element | Guideline |
|---------|-----------|
| Face | Large, expressive, 60%+ of frame |
| Text | Max 3-5 words, 80px+, high contrast |
| Colors | 2-3 bold colors, avoid muted tones |
| Contrast | Test in small size (90x54px) |
| Consistency | Same style across channel for brand recognition |

**Prompt template:**
`"YouTube thumbnail, [SUBJECT] with [EMOTION] expression, bold [COLOR] text overlay '[TITLE]', high contrast, eye-catching, professional"`

## App Store Screenshots

**Strategy:** Tell a story in 3 screenshots
1. Screenshot 1: Core value proposition headline + key feature
2. Screenshot 2: Second most important feature
3. Screenshot 3: Social proof or unique differentiator

**Text overlays:** Use device frames, keep text under 5 words per screenshot, use action-oriented captions

# Video Production Guide

## Shot Types Reference

| Shot | Abbreviation | Use for |
|------|-------------|---------|
| Extreme Wide Shot | EWS | Establishing location, scale |
| Wide Shot | WS | Full scene context |
| Medium Shot | MS | Conversation, action |
| Close-Up | CU | Emotion, product detail |
| Extreme Close-Up | ECU | Texture, emphasis |
| Over-the-Shoulder | OTS | Dialogue, perspective |
| POV | POV | Immersive first-person |
| Tracking | — | Following subject |
| Aerial/Drone | — | Scale, transitions |

## AI Video Prompt Formula

`[Shot type] of [subject], [action], [setting], [lighting], [style], [technical specs]`

**Examples:**
- `"Close-up of hands typing on a keyboard, soft office lighting, cinematic, shallow depth of field, 4K"`
- `"Aerial shot of a city at sunset, golden hour light, slowly pulling back, cinematic color grade, 30fps"`
- `"Tracking shot following a chef plating food, kitchen environment, warm lighting, documentary style, 24fps"`

## Storyboard Creation

**5-scene marketing video structure:**

| Scene | Duration | Content | Shot |
|-------|----------|---------|------|
| Hook | 0-3s | Problem or surprising statement | Close-up |
| Problem | 3-8s | Relatable pain point | Medium shot |
| Solution | 8-20s | Product/service in action | Demo shot |
| Proof | 20-25s | Testimonial or metric | Talking head |
| CTA | 25-30s | Clear next step | Text overlay |

## Talking Head Production

**Equipment checklist:**
- Camera at eye level (not below — unflattering)
- Key light: 45° to the side and slightly above
- Fill light: opposite side, 50% intensity of key
- Background: 3+ feet behind subject to separate
- Microphone: lavalier or shotgun (not built-in)

**Script format:**
```
[HOOK - 5 seconds]
[PROBLEM - 10 seconds]
[SOLUTION - 15 seconds]
[PROOF/DEMO - 20 seconds]
[CTA - 5 seconds]
```

## AI Marketing Video Workflow

1. **Script:** Write 3 versions (long/medium/short: 60s/30s/15s)
2. **Storyboard:** Sketch key frames (use AI image generation for visual reference)
3. **Assets:** Generate backgrounds and b-roll with AI video tools
4. **Record:** Film talking head sections separately
5. **Edit:** Hook first, cut on action, end with CTA

3. **How it works** (40-80s): 3 key steps or features

# Writing Guide

## Technical Blog Post

**Structure:**
1. **Headline** (H1): Problem + Solution promise. Example: "How We Cut API Response Time by 80%"
2. **Introduction** (150 words): Hook with a statistic or problem statement
3. **Body** (800-1500 words): 3-5 sections with H2 headers, code examples, screenshots
4. **Conclusion** (100 words): Summary + next steps or CTA

**Headline formulas:**
| Formula | Example |
|---------|---------|
| How We [Result] | "How We Scaled to 10M Users Without Rewriting Our Stack" |
| [Number] Ways to [Goal] | "5 Ways to Reduce Churn with Better Onboarding" |
| The [Adjective] Guide to [Topic] | "The Complete Guide to Serverless Deployments" |
| Why [Common Belief] is Wrong | "Why REST APIs Are the Wrong Choice for Real-Time Apps" |

## Case Study Writing

**Template:**
```
## [Company Name]: [Result in Numbers]

**Challenge:** [1-2 sentences describing the problem]

**Solution:** [2-3 sentences describing what was implemented]

**Results:**
- [Metric 1]: [Before] → [After] ([% change])
- [Metric 2]: [Before] → [After]
- [Metric 3]: [Before] → [After]

**Quote:** "[Testimonial from decision-maker]" — [Name, Title]
```

**Key elements:** Specific numbers (not "significant improvement"), named stakeholder, before/after comparison, timeline

## Newsletter Curation

**Format for weekly digest:**
1. **Subject line**: "[Number] [topic] reads for [time period]" — max 50 chars
2. **Intro** (2-3 sentences): What's the theme this week?
3. **Items** (3-7 items):
   - Headline (linked)
   - 1-sentence description
   - Why it matters (your take)
4. **CTA**: One question to spark reply engagement

## Press Release Writing

**Structure (inverted pyramid):**
```
[CITY, Date] — [Company] today announced [news in one sentence].

[Paragraph 2: Why this matters, context, numbers]

[Paragraph 3: Quote from executive]

[Paragraph 4: How it works, additional details]

[Paragraph 5: Availability, pricing, next steps]

About [Company]: [2-sentence boilerplate]

Contact: [PR contact info]
```

## SEO Content Brief

**Template:**
| Field | Value |
|-------|-------|
| Target keyword | [primary keyword] |
| Search intent | Informational / Transactional / Navigational |
| Word count | [based on top 3 competitors' average] |
| H1 | [include keyword, max 60 chars] |
| Meta description | [include keyword, 120-160 chars] |
| Internal links | [2-3 relevant existing pages] |
| CTA | [what action should reader take] |

**Keyword placement:**
- H1: once
- First 100 words: once
- H2 headers: 1-2 times (naturally)
- Body: 1-2% density (don't stuff)
- Alt text: once if image relevant
```

# Social Media Guide

## LinkedIn Content

**Post format that performs:**
```
[HOOK — 1-2 lines that create curiosity or bold statement]

[blank line]

[STORY or DATA — 3-5 lines expanding the hook]

[blank line]

[LESSON — 1-2 lines of actionable insight]

[blank line]

[CTA — 1 question to drive comments]
```

**Hook formulas:**
| Type | Example |
|------|---------|
| Contrarian | "Cold emails don't work. Here's what does:" |
| Number | "I sent 500 cold emails last month. Here's what I learned:" |
| Story | "3 years ago I had $0 and an idea. Today we crossed $1M ARR." |
| Mistake | "I made a $50,000 mistake so you don't have to." |

**Best posting times:** Tuesday-Thursday, 7-9am or 5-6pm local time

## Twitter/X Thread Creation

**Thread structure:**
1. **Tweet 1 (hook)**: Bold claim or promise. End with "Thread 🧵"
2. **Tweets 2-7**: One point per tweet, numbered (2/, 3/, etc.)
3. **Tweet 8 (summary)**: "TL;DR:" followed by 3-bullet summary
4. **Tweet 9 (CTA)**: Follow, retweet, or reply prompt

**Engagement rules:**
- Each tweet should stand alone (could be shared individually)
- Use numbers and specifics (not "some" or "many")
- First tweet determines thread performance — test multiple hooks
- Keep individual tweets under 240 characters

## Social Media Carousel (LinkedIn/Instagram)

**10-slide formula:**
| Slide | Content |
|-------|---------|
| 1 (Hook) | Bold headline + visual, promise a transformation |
| 2 (Problem) | Relatable pain point |
| 3-7 (Content) | One tip/insight per slide, numbered |
| 8 (Summary) | Key takeaways in bullets |
| 9 (CTA) | Save, share, follow |
| 10 (Brand) | Logo + website/handle |

**Design rules:**
- Consistent colors and fonts throughout
- Large text (readable in feed preview)
- Simple background (no busy patterns)
- Slide number in corner (helps with navigation)

## AI Social Media Content Batch

**Monthly batch workflow:**
1. **Themes** (1 hour): Define 4 content pillars (e.g., education, behind-scenes, social proof, entertainment)
2. **Ideation** (30 min): Generate 20 ideas per pillar with AI
3. **Writing** (2 hours): Draft all posts in one session
4. **Visuals** (1 hour): Generate or create images for visual posts
5. **Schedule**: Load into scheduling tool, spread across the month

**Repurposing matrix:**
| Original | Repurpose to |
|----------|-------------|
| Blog post | LinkedIn carousel + Twitter thread + 3 tweets |
| Case study | LinkedIn story post + Instagram carousel |
| Webinar | 5 Twitter clips + LinkedIn article |
| Podcast episode | Twitter thread (key insights) + LinkedIn post |

# Product Guide

## Competitor Teardown

**Framework:**

|----------|---------------------|
| Positioning | Who is their ICP? What's their unique value prop? |
| Product | Top 3 features? What's missing? What's overcomplicated? |

**Data sources:**
- G2/Capterra: Real user reviews and ratings
- Job postings: What they're building next
- Changelog/blog: Recent priorities
- Reddit/HN: Unfiltered user sentiment
- Wayback Machine: How positioning evolved

## Customer Persona

**Persona template:**
```
## [Persona Name] — [Job Title]

**Demographics:** [Age range, company size, industry]

**Goals:**
- Primary: [What they're trying to achieve]
- Secondary: [Supporting goals]

**Pain Points:**
- [Pain 1: specific, not vague]
- [Pain 2]
- [Pain 3]
