Frontmatter:
  name: tools-ui
  description: "React patterns to visualize AI tool call lifecycle: pending, running, approval, result."
  allowed-tools: []
  disable-model-invocation: false

Conteudo (guia para visualizar o lifecycle de tool calls):
- Secao "Tool Call Lifecycle": estados — pending (solicitado pelo LLM) → awaiting-approval (se human-in-the-loop) → running (executando) → success (resultado disponivel) → error (falha)
- Secao "ToolCallCard Component": props interface — { name, input, status, output?, error? }; renderizar diferente por status
- Secao "Status Icons": icones/badges por status (spinner para running, check verde para success, X vermelho para error)
- Secao "Human Approval UI": modal ou inline com botoes Approve/Deny quando status=awaiting-approval
- Secao "Input/Output Display": JSON.stringify(input, null, 2) em bloco pre formatado; truncar outputs longos
- Secao "Tool List Sidebar": lista de ferramentas disponiveis com descricao; highlight durante execucao ativa
- Secao "Example: Claude tool_use": como parsear o campo tool_use do response Claude e renderizar
- Secao "Example: OpenAI function_call": como parsear function_call/tool_calls e renderizar
- Tabela de formato de tool calls por provider (Claude, OpenAI-compatible, Gemini)

- Exemplo completo de WidgetRenderer em TypeScript/React (~30 linhas)
