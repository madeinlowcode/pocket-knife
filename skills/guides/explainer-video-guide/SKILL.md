# Explainer Video Creation Guide

## What Makes an Explainer Video Work
An explainer video succeeds when a viewer who knew nothing about your product or concept watches 90 seconds and can immediately describe the problem it solves, who it's for, and what to do next.

---

## Optimal Length
- **60–90 seconds**: Sweet spot for SaaS, product, and service explainers
- **Under 60 sec**: Simple features, single-concept ideas
- **90–120 sec**: Complex B2B products with multiple stakeholders
- Never exceed 2 minutes for a homepage or landing-page explainer

**Word count by duration:**
- 60 sec ≈ 130–150 words of voiceover
- 90 sec ≈ 200–225 words
- 120 sec ≈ 270–300 words

---

## Script Frameworks

### 1. Problem-Solution-Benefit (PSB) — Best for most use cases
```
PROBLEM   (15–20 sec) — Describe the pain the audience feels
SOLUTION  (30–40 sec) — Introduce the product as the answer
BENEFIT   (15–20 sec) — Show the life/work after using it
CTA       (5–10 sec)  — Direct next step
```

**Example beat sheet (90 sec):**
1. Open with relatable pain moment (0–15 sec)
2. Introduce the product by name (15–20 sec)
3. Show how it works — step 1 (20–35 sec)
4. Show how it works — step 2 (35–50 sec)
5. Show how it works — step 3 (50–65 sec)
6. Show the transformed outcome (65–80 sec)
7. CTA with product name repeat (80–90 sec)

### 2. "Meet [Character]" Framework — Best for B2C or consumer apps
```
Introduce relatable character with a specific problem
→ Character discovers the product
→ Step-by-step demonstration through character's eyes
→ Character's life improved
→ CTA
```

### 3. Question Stack Framework — Best for awareness-stage audiences
```
Ask 3 questions the audience is already thinking
→ Acknowledge the frustration
→ Pivot: "What if there were a better way?"
→ Introduce product as answer
→ Demo key features (2–3 max)
→ CTA
```

## Animation Styles

### Whiteboard / Sketch Animation
- **Best for**: Educational content, technical products, B2B
- **Tone**: Intelligent, authoritative, trustworthy
- **Avoid**: If brand is youthful or consumer-focused
- **Tools**: VideoScribe, Doodly

### 2D Motion Graphics
- **Best for**: SaaS products, startup explainers, app demos
- **Tone**: Clean, modern, professional
- **Key technique**: Use your product's actual UI colors
- **Tools**: After Effects, Rive, Lottie

### Character Animation (2D)
- **Best for**: Consumer products, HR/onboarding, healthcare
- **Tone**: Friendly, approachable, human
- **Avoid**: If product is highly technical or enterprise-focused
- **Tools**: Adobe Animate, Vyond, Toonly

### Screen Recording + Motion Graphics Hybrid
- **Best for**: Software products where the UI is the hero
- **Tone**: Functional, credible, direct
- **Technique**: Record actual product, animate callouts and highlights on top

### Live Action + Motion Graphics
- **Best for**: Physical products, services with human teams
- **Tone**: Premium, relatable
- **Cost**: Highest of all styles

## Voiceover Best Practices

### Script Writing for Voiceover
- Write conversationally — read every line aloud before finalizing
- Use contractions (it's, you'll, we're) — they sound natural
- Avoid passive voice ("it can be done" → "you can do it")
- One idea per sentence — never stack two concepts
- Short sentences under pressure, longer ones when explaining nuance

### Casting the Right Voice
| Product Type       | Voice Character           |
|--------------------|---------------------------|
| Enterprise SaaS    | Authoritative, warm male/female |
| Consumer app       | Friendly, upbeat, peer-to-peer  |
| Healthcare/finance | Calm, trustworthy, measured     |
| Creative tool      | Energetic, enthusiastic         |
| Security/legal     | Serious, precise, confident     |

### Recording Quality Standards
- Sample rate: 44.1 kHz / 48 kHz
- Bit depth: 24-bit
- No room reverb — record in treated space or use a closet
- Normalize to -3 dB peak, -16 LUFS RMS (for web)
- Remove breaths between sentences in post

### Pacing Guidelines
- Speak at 130–150 words per minute for clarity
- Pause 0.5 sec at commas, 1 sec at periods
- Emphasize product name and key benefits — slightly slower on those words

## Storyboard-to-Animation Alignment

### Scene Timing Rules
- Each visual scene matches a single spoken concept
- Cut scene when the voiceover transitions to the next idea
- Visual appears 0.5 sec before voiceover mentions it (pre-empt the ear)
- Text callouts reinforce spoken words — never repeat them verbatim

### Animation Do's and Don'ts
| Do                                          | Don't                                 |
|---------------------------------------------|---------------------------------------|
| Use smooth ease-in/ease-out on all motion   | Use linear motion (looks mechanical)  |
| Keep background elements static when VO is critical | Animate everything simultaneously |
| Match animation rhythm to music beat        | Ignore the musical rhythm             |
| Show the product doing the work             | Only show the product logo            |
| Use brand colors throughout                 | Introduce colors not in brand palette |

---

## Music and Sound Design
- Bed music: upbeat but not distracting, -20 dB under voiceover
- UI sounds: subtle confirmation tones when showing product interactions
- Silence: use brief silences (0.3 sec) before key benefit moments — creates emphasis
- Outro music: slight swell as CTA appears

## Production Checklist

### Pre-Production
- [ ] One-sentence core message written and approved
- [ ] Target viewer persona defined
- [ ] Script written, timed, and read aloud
- [ ] Animation style chosen and matches brand
- [ ] Voiceover talent cast (or AI voice selected)
- [ ] Storyboard panels created for every scene change

### Production
- [ ] Style frames (2–3) approved before full animation
- [ ] Voiceover recorded and edited
- [ ] Music selected and licensed

### Post-Production
- [ ] Animation synced to voiceover
- [ ] Music mixed correctly (-20 dB under VO)
- [ ] Captions added (open or closed)
- [ ] Reviewed on mobile screen (most views happen here)
- [ ] Exported: MP4, H.264, 1920×1080 (or 1080×1920 for vertical)
- [ ] CTA is readable within 3 seconds of appearing
