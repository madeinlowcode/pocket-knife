# App Store Screenshot Design Guide

## Core Purpose

App store screenshots are your highest-leverage marketing asset. They must communicate value instantly, establish trust visually, and compel a download decision — often within 3 seconds of scanning.

---

## Screenshot Specifications

### Apple App Store
- iPhone 6.9" (required): 1320 x 2868 px
- iPhone 6.5" (required): 1284 x 2778 px
- iPad Pro 13" (if tablet support): 2064 x 2752 px
- Format: PNG or JPEG, no alpha on JPEG
- Max 10 screenshots per device size

### Google Play Store
- Phone: 1080 x 1920 px minimum (16:9 or 9:16)
- 7" Tablet: 1200 x 1920 px
- 10" Tablet: 1600 x 2560 px
- Format: PNG or JPEG, 8-bit
- Max 8 screenshots

---

## Composition Framework

### The 3-Zone Layout
Every screenshot should be divided into three zones:

1. **Headline Zone** (top 20%): Bold benefit statement, 5 words max
2. **UI Zone** (middle 60%): Actual app interface or device mockup
3. **Support Zone** (bottom 20%): Reinforcing copy or secondary visual

### Orientation Strategy
- **Portrait**: Better for utility apps, productivity, social
- **Landscape**: Better for games, media players, content consumption
- Match your app's primary usage orientation

---

## The First Screenshot Rule

The first screenshot is the only one visible in search results on iOS. It carries 70% of the conversion weight.

Checklist for screenshot #1:
- [ ] Primary value proposition visible without reading
- [ ] Brand color prominent and recognizable
- [ ] Zero ambiguity about what the app does
- [ ] Hero action or hero state of the app shown
- [ ] Emotional appeal (joy, relief, productivity, excitement)

---

## Text Overlay Best Practices

### Typography Rules
- Use a maximum of 2 font weights per screenshot
- Minimum body text: 28px at 1x (renders at 14px on device thumbnails)
- Headline minimum: 48px at 1x
- Line height: 1.2–1.4x for headlines, 1.5x for body
- All-caps works for 1–3 word headlines only

### Copy Frameworks
- **Benefit headline**: "Meditate in 5 minutes" not "Guided Meditation App"
- **Problem/Solution**: "Can't sleep? Wind down in minutes."
- **Social proof**: "Join 2M people who sleep better"
- **Feature callout**: "Auto-categorizes expenses" with arrow pointing to UI

### Text Placement Checklist
- [ ] Text does not overlap critical UI elements
- [ ] Contrast ratio minimum 4.5:1 against background
- [ ] Text safe zones respected (avoid edges within 40px)
- [ ] Works legible at thumbnail size (1/3 of full size)

---

## Device Mockup Guidelines

### When to Use Mockups
- Always use device frames for lifestyle/consumer apps
- Skip frames for pure utility/B2B apps (full-bleed UI converts better)
- Never mix device generations in the same set

### Mockup Composition
- Device should occupy 60–75% of the screenshot height
- Tilt angle: 0° (straight) or 5–15° (subtle dynamic angle)
- Avoid extreme perspective — distorts UI readability
- Shadow under device: soft, 20–30% opacity
- One device per screenshot unless comparison is the point

### Background Treatment
- **Solid color**: Clean, professional — works for utility apps
- **Gradient**: Adds depth — use complementary brand colors
- **Lifestyle photo**: Aspirational — works for wellness, fitness, travel
- **Pattern/texture**: Distinctive — avoid busy patterns that compete with UI

---

## Screenshot Sequence Strategy

### Recommended 5-Screenshot Arc
1. **Hook**: Primary value proposition + hero UI state
2. **Problem solved**: Show the pain point being eliminated
3. **Key feature #1**: Most differentiating capability
4. **Key feature #2**: Second strongest feature
5. **Social proof / CTA**: Ratings, awards, or "Download free"

### 8-10 Screenshot Arc (Extended)
Add between steps 3–4:
- Personalization or customization feature
- Delight moment or unexpected capability
- Integration or ecosystem benefit

---

## Color and Visual Consistency

### Consistency Rules
- Use identical background color family across all screenshots
- Same font family throughout the set
- Same device mockup across all frames
- Consistent text placement zone (top vs. bottom)

### Color Psychology by App Category
- **Finance/Productivity**: Dark navy, slate, forest green — trust, focus
- **Health/Wellness**: Soft greens, warm neutrals, sky blue — calm, growth
- **Social/Entertainment**: Vibrant primaries, bold contrast — energy, fun
- **Utility/Tools**: Clean whites, grays — clarity, precision

## A/B Testing Framework

### What to Test First
Test in this priority order — highest impact to lowest:

1. **First screenshot only** — the 70% decision maker
2. **Background color** — can shift conversion 10–25%
3. **Headline copy** — benefit vs. feature framing
4. **Portrait vs. landscape orientation**
5. **With device frame vs. without**

### Testing Protocol
- Run tests for minimum 7 days (capture weekly usage patterns)
- Need minimum 1,000 impressions per variant for significance
- Change one variable at a time
- Document results: date, variant, impressions, conversion rate, install rate

### Metrics to Track
- **Impression-to-page-view rate**: Are screenshots driving clicks?
- **Page-view-to-install rate**: Are screenshots converting to downloads?
- **Category benchmark**: Compare against app store average (usually 3–6%)

## Common Mistakes to Avoid

- Showing login/onboarding screens — show the value state, not the entry
- Using actual user data (privacy issue + not aspirational)
- Tiny, unreadable UI text — if it can't be read at thumbnail, cut it
- Inconsistent screenshot style that looks like multiple designers worked on it
- Ignoring localization — text in screenshots needs translated versions
- Skipping the iPad screenshots (Apple penalizes apps without them)

---

## Localization Checklist

- [ ] Text length accounts for German/French expansion (30% longer)
- [ ] RTL languages (Arabic, Hebrew) require mirrored layouts
- [ ] Currency/date formats match locale
- [ ] Cultural color meanings checked (white = mourning in some markets)
- [ ] Separate screenshot sets for top 5 markets minimum

---

## Final Quality Checklist

- [ ] Correct dimensions for each required device size
- [ ] No copyrighted UI from third-party apps shown
- [ ] Status bar shows clean state (full battery, strong signal, clean time)
- [ ] App icon visible in at least one screenshot
- [ ] All screenshots reviewed at actual thumbnail display size
- [ ] Exported at correct color profile (sRGB)
